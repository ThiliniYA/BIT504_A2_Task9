This project was created to showcase practical experience with:

## 🔄 Version Control Workflow

### **1. Repository Creation**
A new public GitHub repository named **BIT504_A2_Task9** was created with no initial files.

### **2. Java Project Setup**
A new Java project was created in Eclipse and connected to the Git repository.

### **3. Initial Commit**
The first version of the Java code was staged, committed, and pushed to GitHub.

### **4. Branching**
Two feature branches were created:

- `feature-update1`
- `feature-update2`

Each branch contained its own code updates and commits.

### **5. Pull Requests**
Pull requests were created for both feature branches to merge changes into the `main` branch.

### **6. Merging**
Both pull requests were reviewed and successfully merged into the main branch using GitHub’s merge tools.
